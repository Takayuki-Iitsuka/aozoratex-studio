配色設計ガイド

本ページは、AozoraTeX Studio における配色設計の方針・基準・運用手順をまとめた詳細ガイドです。UI での選択結果と生成物の見え方を一致させることを目的にしています。

1. 基本方針

  • 可読性を最優先し、背景色と文字色のコントラストを確保する。
  • 配色データは static/color-palettes.json を単一ソースとして管理する。
  • Web UI、配色一覧、API のすべてで同一データを参照し整合性を維持する。

2. モード定義

mode想定利用設計意図
light日中読書明背景 + 暗文字で高い視認性を確保
dark夜間読書暗背景 + 明文字で眩しさを抑制
intermediate長時間読書中明度配色で眼精疲労を軽減
presetテーマ読書セピア等の意図的な体験設計

3. コントラスト基準

  • 候補生成時はコントラスト比 4.5:1 以上を基準とする。
  • 長時間読書向けでは、必要に応じて純白/純黒を避けた中間色を採用する。
  • 同系色の過密表示を避けるため、一覧表示は上限件数を制御する。

4. 運用手順

  1. 配色候補を colors.html で確認する。
  2. Web UI で対象デバイスに適用し、本文サンプルで視認性を確認する。
  3. 採用配色を設定保存し、必要に応じてモード別色（background_color_* / text_color_*）を更新する。

5. カスタマイズ時の注意

  • JSON へ色を追加する場合は HEX 大文字（例: #F5F5DC）に統一する。
  • 背景と文字のセットで検証し、単色だけで判断しない。
  • フォント変更時は同じ配色でも見え方が変わるため、再確認を行う。
