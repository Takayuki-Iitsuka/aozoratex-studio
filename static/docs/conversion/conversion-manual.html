変換・文字コード・LaTeX設定ガイド

本ページは、入力HTMLの読み込みから TEX/PDF 出力までの変換処理を、実装単位で追える詳細ガイドです。障害調査や変換ロジック改修時の基準として利用してください。

1. 変換パイプライン

  1. 入力ファイル読込（ローカルファイルのみ）
  2. 文字コード判定（宣言検出 + フォールバック）
  3. HTML パース（BeautifulSoup）
  4. 本文ノードを LaTeX 断片へ変換
  5. テンプレートへ埋め込み TEX 出力
  6. 必要時に latexmk で PDF コンパイル

2. 文字コード運用

  • 候補順序: 指定値 → 宣言値 → utf-8-sig → utf-8 → cp932 → shift_jis
  • 内部表現は UTF-8 に統一する。
  • 判定が不安定な入力は --encoding で固定する。

3. HTML から LaTeX への主な変換

HTML要素変換先補足
<ruby>\ltjruby{親}{ルビ}rb/rt を再帰変換
<img class="gaiji">\AozoraGaiji{...}{...}既知外字は Unicode 置換
<h1>〜<h6>見出しコマンドタイトル/中見出しを分離
テキストノードエスケープ文字列LaTeX 特殊文字を保護

4. CLI 実行例

# 単体変換
.venv\Scripts\python.exe -m src.aozoratex data/1567_14913.html --device iphone --mode light

# ディレクトリ一括変換
.venv\Scripts\python.exe -m src.aozoratex data --device ipad_landscape --out out/ipad_landscape

# 文字コード明示
.venv\Scripts\python.exe -m src.aozoratex data/old.html --encoding cp932 --device android

5. Web API での PDF 生成

  • POST /api/generate に compile_pdf=true を指定すると TEX 生成後に PDF まで処理する。
  • compile_pdf=false では TEX のみ生成し、変換ロジック確認を高速化できる。
  • 失敗時はレスポンスの error と compile_log を優先的に確認する。

6. 調査時チェックリスト

  1. 入力ファイルの存在確認（data/ 配下か）
  2. 設定ロード確認（/api/settings）
  3. 変換ログ確認（out/ 配下）
  4. コンパイルツール確認（latexmk, lualatex）
