変換・文字コード・LaTeX設定ガイド

このページは「青空文庫のHTMLがどのようにしてPDFになるのか」を、手順ごとにわかりやすく説明したものです。
変換でうまくいかないときの調査や、仕組みを理解したいときに読んでください。

1. 全体の流れ（変換パイプライン）

HTML → PDF になるまでの大まかな手順は以下の通りです。

  1. 入力ファイルを読む（BeautifulSoupでHTML解析、文字コードの自動判定）
  2. 作品情報の抽出（タイトル、著者名、底本情報などのメタデータを取得）
  3. 本文をLaTeXの命令に変換（ルビや外字、傍点などの青空文庫特有の記法をLaTeXマクロに変換）
  4. 装飾・デバイス設定の適用（デバイスごとのサイズや段組、和紙背景・フレーム・表紙テクスチャ等の装飾を反映）
  5. テンプレートに埋め込んで .tex ファイルを作成（out/work/{device_id}/ フォルダに保存）
  6. LuaLaTeX でコンパイル（必要なら latexmk を使用し PDF を自動生成し、out/pdf/{device_id}/ フォルダに配置）

2. 文字コードの扱い方

青空文庫の古いファイルは文字コードがバラバラです。AozoraTeXは以下のような優先順位で判断します。

  • あなたが --encoding で指定したもの（CLIの場合）
  • HTMLの中に書かれている宣言（metaタグなど）
  • utf-8-sig → utf-8 → cp932 → shift_jis の順でフォールバック

変換時の文字化けへの対処：
CLIでは --encoding cp932 のように明示的に指定してみてください。Web UIからのアップロード時は自動判定が行われます。

3. HTMLのどの部分がどう変換されるか

HTMLの要素 LaTeXでの変換結果 補足
<ruby> / <rb> / <rt> \ltjruby{親}{ルビ} ルビ（ふりがな）をLuaTeX-jaの命令で正しく表示
<img class="gaiji"> \AozoraGaiji{...} / Unicode 外字（特殊な漢字）をUnicodeに置き換え、または独自マクロ化
<em class="sesame_dot"> など \kenten{テキスト} 傍点や圏点をLaTeXのルビ/圏点マクロで表現
<h1>〜<h6> \section* などの見出し命令 タイトルと本文の見出しを自動で分けて変換
普通のテキスト エスケープ処理された文字列 LaTeXで特別な意味を持つ文字（& % $ # _ { } \ など）を安全にエスケープ

4. 装飾とレンダリングモード

最新の AozoraTeX Studio では、美しいPDFを作るための背景描画モードが用意されています。

  • レンダリングモード（tikz / image）
    ベクター描画の tikz（くっきり綺麗だがコンパイルが少し遅い）と、画像貼り付けの image（コンパイルが速い）を選べます。
  • 装飾オプション
    表紙テクスチャ、和紙背景、装飾フレームなどの有無を config/global_settings.custom.ini または Web UI からコントロール可能です。

5. CLI（コマンドライン）での実行例

# 1つのファイルを変換する
.venv\Scripts\python.exe -m src.aozoratex_cli data/1567_14913.html --device iphone

# フォルダの中の全部を一括変換し、背景や文字色を変える
.venv\Scripts\python.exe -m src.aozoratex_cli data --device ipad --bg-color "#FDF6E3" --fg-color "#657B83"

# 装飾を付けて変換する
.venv\Scripts\python.exe -m src.aozoratex_cli data/1567_14913.html --main-washi --main-frame --cover-texture

6. Web画面（Next.js）からPDFを作る場合

  • 画面でファイルを選んで「PDFを生成する」ボタンを押すと、内部で /api/generate API が呼ばれます。
  • APIは compile_pdf=true（既定）で CLI 相当のロジックを呼び出し、.tex を作った後にPDFまで完成させます。
  • UIで選んだデバイス（iphone, ipad など）や、背景色・文字色・段組・装飾オプションなどの状態は、APIリクエスト時に JSON として送信され、変換処理にすべて反映されます。

7. 変換がうまくいかないときのチェックリスト

  1. ファイルが data/ フォルダ等、アクセス可能な場所にあるか確認（CLIの場合）
  2. Web画面の場合：設定が正しく読み込まれているか（右上のギアアイコンから設定を確認）
  3. out/work/<デバイス名>/ フォルダの中の .log ファイルや、.tex ファイルの中身を見る
  4. TeXがインストールされているか（lualatex --version で確認）
