変換・文字コード・LaTeX設定ガイド

このページは「青空文庫のHTMLがどのようにしてPDFになるのか」を、手順ごとにわかりやすく説明したものです。
変換でうまくいかないときの調査や、仕組みを理解したいときに読んでください。

1. 全体の流れ（変換パイプライン）

HTML → PDF になるまでの大まかな手順は以下の通りです。

  1. 入力ファイルを読む（data/ フォルダ内のHTMLだけ対応）
  2. 文字コードを判定する
  3. HTMLを解析する（BeautifulSoup）
  4. 本文をLaTeXの命令に変換する
  5. テンプレートに埋め込んで .tex ファイルを作成
  6. 必要なら latexmk で PDFを自動生成

2. 文字コードの扱い方

青空文庫の古いファイルは文字コードがバラバラです。AozoraTeXは以下のような優先順位で判断します。

  • あなたが --encoding で指定したもの
  • HTMLの中に書かれている宣言（metaタグなど）
  • utf-8-sig → utf-8 → cp932 → shift_jis の順で試す

変換に失敗したときの対処：
--encoding cp932 のように明示的に指定してみてください。

3. HTMLのどの部分がどう変換されるか

HTMLの要素 LaTeXでの変換結果 補足
<ruby> \ltjruby{親}{ルビ} ルビ（ふりがな）を正しく表示
<img class="gaiji"> \AozoraGaiji{...} 外字（特殊な漢字）をUnicodeに置き換え
<h1>〜<h6> 見出し用のLaTeX命令 タイトルと本文の見出しを自動で分ける
普通のテキスト エスケープ処理された文字列 LaTeXで特別な意味を持つ文字（& % $ など）を安全にする

4. CLI（コマンドライン）での実行例

# 1つのファイルを変換する
.venv\Scripts\python.exe -m src.aozoratex_cli data/1567_14913.html --device smart

# フォルダの中の全部を一括変換
.venv\Scripts\python.exe -m src.aozoratex_cli data --device tablet --out out/tablet

# 文字コードが古いファイルの場合
.venv\Scripts\python.exe -m src.aozoratex_cli data/old.html --encoding cp932 --device smart

5. Web画面（Next.js）からPDFを作る場合

  • 画面でファイルを選んで「PDF化」ボタンを押すと、内部で /api/generate が呼ばれます。
  • compile_pdf=true（既定）だと、.tex を作った後にPDFまで完成させます。
  • 変換だけ確認したいときは compile_pdf=false にすると速く終わります。

6. 変換がうまくいかないときのチェックリスト

  1. ファイルが data/ フォルダの中にあるか確認
  2. Web画面の場合：設定が正しく読み込まれているか（「設定を保存する」ボタン）
  3. out/ フォルダの中のログやエラーメッセージを見る
  4. TeXがインストールされているか（lualatex --version で確認）
