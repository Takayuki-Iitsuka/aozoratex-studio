プロジェクト概要・運用ガイド

このページは「AozoraTeX Studioをどう使えばいいか」「どう保守すればいいか」をまとめたものです。
新しく参加する人向けのオンボーディング資料としても、日常の運用を見直すときにも役立ちます。

1. プロジェクト目的

  • 青空文庫 HTML/XHTML を LuaLaTeX 向け TEX へ変換し、デバイス最適化された PDF 読書体験を提供する。
  • Web UI と CLI の二経路で同じ設定体系（config/*.ini）を利用し、運用差分を最小化する。
  • 設定・配色・フォント・装飾を再利用可能な形で保存し、再現性の高い出力を維持する。

2. 現在の仕組み（アーキテクチャ）

簡単に言うと、以下の部品でできています：

  • 変換の心臓部： src/aozoratex.py （HTML → LaTeX変換）
  • 画面と操作： Next.js 15（src/app/page.tsx + Tailwind）
  • Pythonとの橋渡し： src/api_bridge.py + src/server_services.py
  • 設定の管理： src/settings_store.py + config/*.ini
  • 入力と出力： data/（入力HTML） → out/（生成物）

3. 基本的な運用ルール

  1. 普段はWeb画面を使う。大量処理やテストはCLIを使う。
  2. 設定の優先順位： default.ini → custom.ini で上書き。
  3. 出力は全部 out/ にまとめる。調査しやすくするため。
  4. ドキュメントは常に最新に。変更したら説明も一緒に更新。

4. 品質を保つためのチェック

  • Pythonの文法チェック：python -m compileall src tools
  • 変換テスト：サンプルHTMLでCLI実行
  • 画面の動作確認：設定取得・デバイス一覧などが正しく返ってくるか

5. カスタマイズ運用

  • 配色追加: static/color-palettes.json を編集
  • デバイス調整: config/device_settings.custom.ini に上書き値を追加
  • 既定変更: config/*.default.ini のみ編集し、影響範囲を明示する
  • 装飾制御: Web UI または CLI オプション（--main-washi など）で反映
