プロジェクト概要・運用ガイド

このページは「AozoraTeX Studioをどう使えばいいか」「どう保守すればいいか」をまとめたものです。
新しく参加する人向けのオンボーディング資料としても、日常の運用を見直すときにも役立ちます。
現在、システムは Next.js 16 (App Router) を中心としたモダンなスタックと、安定した Python 変換エンジンのハイブリッド構成で稼働しています。

1. プロジェクト目的

  • 青空文庫 HTML/XHTML を LuaLaTeX 向け TEX へ変換し、デバイスに最適化された高品質な PDF 読書体験を提供する。
  • Next.js ベースの Web UI と Python CLI の二経路で同じ設定体系（config/*.ini および JSONベースの設定）を利用し、運用差分を最小化する。
  • 設定・配色・フォント・装飾を再利用可能な形で保存し、再現性の高い出力を維持する。
  • Pydantic を用いた堅牢な設定値検証により、安全な実行環境を提供する。

2. 現在の仕組み（アーキテクチャ）

システムは以下のコンポーネントが密接に連携して動作しています：

  • フロントエンド (UI/UX)： Next.js 16、React 19、Tailwind CSS v4 で構築されたリッチなクライアントUI (src/app/page.tsx など)。Framer Motion (motion) による滑らかなアニメーションを備えています。
  • バックエンド/API ブリッジ： Node.js (Next.js API Routes) から Python プロセスを非同期で呼び出す src/api_bridge.py および src/server_fastapi.py などの橋渡し層。
  • 変換の心臓部 (Core)： src/aozoratex.py を中心とした Python プログラム。BeautifulSoup4 と lxml を用いた高速で高度な HTML 解析と、文字列ベースの LaTeX テンプレート生成を担当します。
  • 設定の管理： src/settings_store.py と Pydantic による設定検証。config/*.ini から安全に設定を読み込み・バリデーションします。
  • 入力と出力： data/ ディレクトリに配置した青空文庫形式HTMLを入力とし、生成物 (TEX, PDF) は out/ に一元的に出力されます。

3. 基本的な運用ルール

  1. 普段はWeb画面を使う。Next.js 開発サーバー (npm run dev) を起動し、ブラウザ上で直感的に操作します。
  2. 大量処理やテストはCLIを使う。uv run aozoratex や python -m src.aozoratex_cli を利用して高速なバッチ処理を行います。
  3. 設定の優先順位： config/default.ini をベースとし、config/custom.ini で環境やユーザーごとの設定を上書きします。
  4. 出力は全部 out/ にまとめる。TEXソースや中間ファイル、最終的なPDFを隔離し、Gitリポジトリをクリーンに保つためです。
  5. 依存関係の管理： Node.js 側は package.json、Python 側は pyproject.toml (uv推奨) で厳格に管理します。

4. 品質を保つためのチェック

  • TypeScript / フロントエンド： npm run lint (ESLint v9), npm run format:check (Prettier) による静的解析。
  • Python コード： ruff による高速な Lint、mypy による型チェック (uv run ruff check . / uv run mypy src)。
  • 変換テスト： pytest を用いたユニットテスト。サンプルHTMLを用いたCLI実行による回帰テスト。
  • 画面の動作確認： Next.js UI 上で、設定の取得・デバイス一覧の取得・PDF生成フローが正しく機能するか確認します。

5. カスタマイズ運用

  • 配色追加: static/color-palettes.json を編集し、Web UI から選べるテーマを増やすことができます。
  • デバイス調整: config/device_settings.custom.ini に各デバイス (Kindle, iPadなど) 向けの余白やフォントサイズ設定を追加します。
  • 既定値の変更: config/default.ini 等のデフォルト値ファイルは慎重に編集し、影響範囲を明確にします。
  • 装飾・テクスチャ制御: Web UI または CLI オプション（--main-washi など）で和紙背景などの装飾を動的に切り替えます。
