プロジェクト概要・運用

本ページは、AozoraTeX Studio の 運用基準 と 保守基準 をまとめた詳細ガイドです。新規参加者のオンボーディングと、既存運用の見直しの両方に利用できます。

1. プロジェクト目的

  • 青空文庫 HTML/XHTML を LuaLaTeX 向け TEX へ変換し、デバイス最適化された PDF 読書体験を提供する。
  • Web UI と CLI の二経路で同じ設定体系（config/*.ini）を利用し、運用差分を最小化する。
  • 設定・配色・フォント・装飾を再利用可能な形で保存し、再現性の高い出力を維持する。

2. 現行アーキテクチャ

  • 変換本体: src/aozoratex.py
  • Web API / UI サーバー: src/aozora_server.py
  • 設定ストア: src/settings_store.py
  • UI: static/index.html, static/js/index.js, static/css/index.css
  • 設定ファイル: config/global_settings.*.ini, config/device_settings.*.ini
  • 入力/出力: data/, out/

3. 運用原則

  1. 実行パスを統一する。通常は Web UI、バッチ変換や検証は CLI を使用する。
  2. 設定は default -> custom の優先順を維持する。
  3. 出力物と中間物を out/ 配下に集約し、調査容易性を確保する。
  4. 仕様変更時は対応ドキュメントを同時更新し、コードと説明の乖離を防ぐ。

4. 品質向上フロー

  • 静的確認: python -m compileall src tools
  • 変換確認: サンプル HTML を対象に CLI で TEX 生成
  • API確認: /health, /api/devices, /api/settings の応答確認
  • ドキュメント確認: 旧用語・旧設定記述の残存チェック

5. カスタマイズ運用

  • 配色追加: static/color-palettes.json を編集
  • デバイス調整: config/device_settings.custom.ini に上書き値を追加
  • 既定変更: config/*.default.ini のみ編集し、影響範囲を明示する
  • 装飾制御: Web UI または CLI オプション（--main-washi など）で反映
