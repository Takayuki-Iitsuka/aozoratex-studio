開発者向け説明書

AozoraTeX Studio の現行実装（src/ + config/*.ini + static/）を前提にした運用ガイドです。

1. 目的

  • 青空文庫 HTML/XHTML を LuaLaTeX 向け .tex へ変換する。
  • Web UI と CLI の両方で、デバイス別レイアウトと配色を管理する。
  • 設定変更を config/*.default.ini / config/*.custom.ini に分離して保守性を高める。

2. 前提環境

項目 要件 補足
Python 3.10 以上 仮想環境（.venv）推奨
TeX Live LuaLaTeX が利用可能 PDF 生成には latexmk と lualatex が必要
pip 依存 requirements.txt beautifulsoup4, lxml, flask, markdown

3. クイックスタート

# 仮想環境作成（初回）
python -m venv .venv

# 仮想環境有効化
.venv\Scripts\Activate.ps1

# 依存導入
pip install -r requirements.txt

# Web UI 起動
python -m src.aozora_server

ブラウザで http://localhost:5000 を開きます。

4. ディレクトリ構成（現行）

aozoratex-studio/
├── src/
│   ├── aozoratex.py              # HTML/XHTML -> .tex 変換本体
│   ├── aozora_server.py          # Flask API / Web UI サーバー
│   ├── aozoratex_generate.py     # 生成処理の薄いファサード
│   ├── settings_store.py         # 設定ロード/保存/移行
│   └── templates/
│       └── washi_texture.lua
├── config/
│   ├── global_settings.default.ini
│   ├── global_settings.custom.ini
│   ├── device_settings.default.ini
│   └── device_settings.custom.ini
├── static/
│   ├── index.html                # Web UI
│   ├── css/index.css
│   ├── js/index.js
│   └── docs/                     # ブラウザ閲覧用ドキュメント
├── data/                         # 入力HTML/XHTML
├── out/                          # 出力物（.tex / .pdf / ログ）
└── tools/                        # 補助スクリプト

5. Web UI 運用

  1. 入力ファイルを選択
  2. 出力デバイスを選択
  3. カラーモードと配色を選択
  4. フォントと装飾オプションを選択
  5. 単体または一括で PDF 生成

設定保存は /api/settings、初期化は /api/settings/reset を利用します。

6. CLI 運用

python -m src.aozoratex data/92_14545.html --device iphone --mode light --out out

CLI/Web/Make いずれも、生成先は out/work/<device>/（TEX と中間生成物）および out/pdf/<device>/（PDF）へ統一します。

device TEX と中間生成物 PDF
iphone out/work/iphone/ out/pdf/iphone/
android out/work/android/ out/pdf/android/
ipad out/work/ipad/ out/pdf/ipad/
ipad_landscape out/work/ipad_landscape/ out/pdf/ipad_landscape/
pc out/work/pc/ out/pdf/pc/

出力ファイル名は 元ファイル名_タイトル_作者名 形式です。

主要オプション

オプション 説明
--device iphone/android/ipad/ipad_landscape/pc
--mode light/dark/intermediate
--bg-color, --fg-color HEX 色を直接上書き
--save-settings 現在設定を *.custom.ini へ保存
--reset-settings *.custom.ini を初期化
--main-washi, --main-frame, --cover-texture 装飾オプションの明示制御

7. デバイスプロファイル

device サイズ(mm) 備考
iphone 65 x 140 CLI 既定値
android 70 x 155.5 Android 現代型
ipad 158 x 227 単段組
ipad_landscape 227 x 158 二段組
pc 210 x 297 A4

8. 設定ファイル設計

  • global_settings.default.ini: プロジェクト標準値
  • global_settings.custom.ini: ユーザー上書き
  • device_settings.default.ini: デバイス標準値
  • device_settings.custom.ini: デバイス上書き

読み込み順は default -> custom で、実行時は config/*.ini のみを参照します。

9. API 一覧（抜粋）

Endpoint 用途
GET /api/data-files 入力候補一覧
GET /api/devices デバイス設定取得
GET /api/settings 現在設定取得
POST /api/settings 設定保存
POST /api/settings/reset 設定初期化
POST /api/generate 単体生成
POST /api/generate-batch 一括生成
GET /health ヘルスチェック

10. カスタマイズポイント

  • 配色候補: static/color-palettes.json
  • UI 挙動: static/js/index.js
  • UI 見た目: static/css/index.css
  • 組版ロジック: src/aozoratex.py
  • 設定バリデーション/移行: src/settings_store.py

11. トラブルシューティング

依存導入に失敗する

python -m pip install --upgrade pip
pip install -r requirements.txt

PDF が生成されない

  • latexmk と lualatex の PATH を確認する。
  • まず compile_pdf=false で .tex 生成可否を確認する。
  • out/work/<device>/ のログを確認する。

フォントが反映されない

  • フォント名を config/global_settings.custom.ini の font_family に設定する。
  • Web UI の「フォント一覧を再取得」を実行する。
