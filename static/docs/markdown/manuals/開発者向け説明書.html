開発者向け説明書

青空文庫の HTML から LuaLaTeX を使って縦書き PDF を生成するツールです。 Flask による Web UI と Python CLI の 2 通りの方法で動作します。

注意: この文書には旧構成の記述が一部残っています。最新の構成・実行手順は README.md を優先してください。

1. システム要件

ソフトウェア バージョン 備考
Python 3.10 以上 venv を推奨
TeX Live 2023 以上（LuaHBTeX 含む） lualatex コマンドが PATH に通っている必要があります
pip パッケージ requirements.txt 参照 beautifulsoup4, lxml, flask
OS Windows 10/11 パス区切りと cp932 を考慮した実装になっています。Linux/macOS でも動作しますが、Makefile の一部コマンドは Windows 専用です。

日本語フォント

LuaLaTeX の縦書きには日本語フォントが必要です。次のいずれかをシステムにインストールしておいてください。

  • 游明朝（Yu Mincho） — Windows 10/11 に標準搭載。settings.ini のデフォルトに指定されています。
  • IPAMincho — IPA フォントパッケージから入手できます。游明朝が使えない環境でのフォールバックです。
  • ヒラギノ明朝 — macOS 標準。

TeX Live 確認コマンド

lualatex --version

出力例: This is LuaHBTeX, Version 1.18.0 ...

4. アプリの実行方法

4.1 Web UI（Flask サーバー）

ブラウザ上でHTMLファイル・デバイス・配色を選択して PDF を生成する方法です。

# 仮想環境を有効化してからサーバーを起動
.venv\Scripts\Activate.ps1
python -m src.aozora_server

または:

make server

起動後、ブラウザで http://localhost:5000 を開いてください。

操作の流れ:

  1. data/ フォルダ内の HTML ファイルを選択
  2. 出力デバイス（iPhone / Android / iPad / PC）を選択
  3. 配色モード（Light / Dark / Intermediate または カスタムカラー）を選択
  4. 「Generate」ボタンを押す
  5. .tex ファイルが out/session/work/ に生成され、lualatex が利用可能な場合は PDF も自動生成されます

各エンドポイント:

URL 内容
http://localhost:5000/ メイン Web UI
http://localhost:5000/docs HTML ドキュメント一覧
http://localhost:5000/colors.html 配色サンプル（基本）
http://localhost:5000/colors_200.html 旧URL（/colors.html へリダイレクト）
http://localhost:5000/device-paper-size-map.html デバイス用紙サイズ対応表
http://localhost:5000/health ヘルスチェック（JSON）

4.2 CLI（コマンドライン）

.tex ファイルを直接生成したい場合に使います。PDF への変換は別途 lualatex コマンドが必要です。

基本構文

python -m src.aozoratex <入力HTMLファイルまたはディレクトリ> [オプション]

オプション一覧

オプション デフォルト 説明
source （必須） 入力 HTML/XHTML ファイルパス、またはディレクトリ
--out <DIR> out 出力ディレクトリ
--device <DEVICE> iphone 出力デバイス（下表参照）
--mode <MODE> settings.ini の値 カラーモード: light / dark / intermediate
--bg-color <HEX> modes の設定値 背景色（例: #FDF6E3）。--mode より優先。
--fg-color <HEX> modes の設定値 文字色（例: #657B83）。--mode より優先。
--font <FONT> settings.ini の値 日本語フォント名（例: IPAMincho）。settings.ini より優先。
--encoding <ENC> 自動検出 入力ファイルのエンコーディング（例: cp932）
--parser <PARSER> lxml BeautifulSoup のパーサー: lxml / html.parser / lxml-xml
--verbose オフ 詳細ログをコンソールに出力

デバイス一覧

--device 値 用紙サイズ 説明
iphone 65 × 140 mm iPhone 11 実寸（デフォルト）
android 70 × 155.5 mm Android 現代型（9:20）
android_legacy 70 × 124 mm Android 旧型（9:16）
ipad 158 × 227 mm iPad 縦向き（ipad_portrait の別名）
ipad_portrait 158 × 227 mm iPad 縦向き（単段組）
ipad_landscape 227 × 158 mm iPad 横向き（二段組推奨）
pc 210 × 297 mm A4
smartphone 70 × 140 mm 汎用スマートフォン（互換性用）

実行例

# iPhone 用・ライトモード
python -m src.aozoratex data/1567_14913.html --device iphone --mode light

# Android 用・ダークモード・出力先を指定
python -m src.aozoratex data/1567_14913.html --device android --mode dark --out out/android

# カスタムカラーで PC 用
python -m src.aozoratex data/1567_14913.html --device pc --bg-color "#FDF6E3" --fg-color "#3C3C3C"

# data/ フォルダ内の全 HTML を一括変換
python -m src.aozoratex data/ --device iphone --mode light --verbose

# Shift_JIS エンコードのファイルを指定
python -m src.aozoratex data/old_book.html --encoding cp932 --device iphone

5. LuaLaTeX コンパイルコマンド

src/aozoratex.py は .tex ファイルを生成するだけです。PDF にするには lualatex を 2 回実行する必要があります（相互参照・ページ番号の解決のため）。

基本コマンド

# out/ フォルダに移動して PDF を生成
cd out
lualatex -interaction=nonstopmode 1567_14913.tex
lualatex -interaction=nonstopmode 1567_14913.tex  # 2回目（最終ページ番号確定）

オプション説明

オプション 説明
-interaction=nonstopmode エラーが発生しても対話入力を求めず処理を続行（バッチ処理向け）
-halt-on-error 最初のエラーで処理を停止（デバッグ時に有用）
-output-directory <DIR> 出力先ディレクトリを指定
-synctex=1 SyncTeX 情報を生成（エディタとの同期に使用）

別ディレクトリへの出力

lualatex -interaction=nonstopmode -output-directory out out/1567_14913.tex
lualatex -interaction=nonstopmode -output-directory out out/1567_14913.tex

Web UI による自動コンパイル

python -m src.aozora_server を通じて生成する場合、lualatex が PATH に入っていれば自動的に 2 回コンパイルが実行されます。 TeX Live が未インストールの場合は .tex の生成までとなり、PDF の生成はスキップされます。

よく出るコンパイルエラーと対処

エラーメッセージ 原因 対処
! Font ... not found 指定したフォントがインストールされていない settings.ini の font を IPAMincho に変更
! LaTeX Error: File 'jlreq.cls' not found TeX Live の jlreq パッケージが未インストール tlmgr install jlreq
! LaTeX Error: File 'luatexja.sty' not found luatexja パッケージが未インストール tlmgr install luatexja
lualatex が見つかりません PATH に lualatex が通っていない TeX Live のインストールと PATH 設定を確認

8. 処理フロー

    │

  ├── <ruby>      → \ltjruby{親}{ルビ}

    │
    ▼ 出力
<output_dir>/<filename>.tex

9. テンプレート管理（新仕様）

方針

LaTeX テンプレートは latex_templates/ ディレクトリでは管理せず、 src/aozoratex.py 内のテンプレート定数で一元管理します。

この仕様変更により、テンプレート編集と変換ロジックを同一ファイルで保守できるため、 修正時の追跡性と移植性が向上します。

主要テンプレート定数

定数 用途
DEFAULT_FRONTCOVER_TEMPLATE 表紙レイアウト（タイトル・著者）
DEFAULT_TYPESETTING_INFO_TEMPLATE 組版情報ページ
DEFAULT_MAIN_TEXT_TEMPLATE 本文ページ
DEFAULT_COLOPHON_TEMPLATE 奥付ページ
WASHI_TEXTURE_TEMPLATE 和紙背景描画（Lua/TikZ）
COVER_TEXTURE_TEMPLATES 表紙テクスチャバリエーション
MAIN_FRAME_TEMPLATES 本文外周枠バリエーション

レンダリング処理

  • プレースホルダ置換は render_template_block() で実行します。
  • 実際の組み立ては build_tex_file() で行います。
  • 和紙有効時は 表紙・本文・奥付 に背景を適用します。

Web UI テンプレート

Web UI のトップページも Jinja テンプレート方式ではなく、static/index.html を静的配信します。 入力ファイル一覧は GET /api/data-files で取得し、static/js/index.js 側で動的に描画します。

10. Web API リファレンス

src/aozora_server.py (Flask) が提供する REST API の一覧です。

GET エンドポイント

エンドポイント 説明 レスポンス例
GET /api/data-files data/ 内の HTML ファイル一覧 {"files": [{"name": "...", "path": "..."}]}
GET /api/devices デバイス設定一覧 {"iphone": {"width": 65, ...}}
GET /api/colors?mode=light 配色スキーム一覧（mode: light/dark/all） {"mode": "light", "schemes": [...]}
GET /health サーバー稼働確認 {"status": "ok", "time": "..."}

POST エンドポイント

POST /api/generate — 単一ファイル生成

{
    "source": "data/1567_14913.html",
    "device": "iphone",
    "mode": "light",
    "bg_color": "#FFFFFF",
    "fg_color": "#000000",
    "compile_pdf": true
}

レスポンス:

{
    "success": true,
    "tex_file": "out/session/work/1567_14913.tex",
    "pdf_file": "out/session/pdf/1567_14913.pdf",
    "pdf_url": "/out/session/pdf/1567_14913.pdf"
}

POST /api/generate-batch — 複数ファイル一括生成

{
    "device": "iphone",
    "mode": "light",
    "bg_color": "#FFFFFF",
    "fg_color": "#000000",
    "compile_pdf": true,
    "generate_all": true
}

generate_all: true で data/ 内の全ファイルを対象にします。 sources: ["data/xxx.html", ...] で個別指定も可能です。

POST /api/session/organize — セッション出力を整理

out/session/ 直下のファイルを pdf/ と work/ に仕分けします。

POST /api/session/cleanup-nonpdf — 中間ファイル削除

out/session/ 配下の PDF 以外（.tex, .log, .aux など）を削除します。

11. 未使用・保留ファイルの説明

latex_templates/（削除済み）

旧仕様で使用していた外部 LaTeX テンプレートディレクトリは廃止されました。 現在は src/aozoratex.py 内の埋め込みテンプレート定数を使用します。

html_templates/（削除済み）

Web UI は静的配信方式へ移行済みです。 トップページは static/index.html、データ取得は API（/api/data-files）を使用します。

backup/misc/1567_14913.ltjruby

過去のビルドで生成されたルビ情報の中間ファイルです。make clean の対象ですが、バックアップフォルダ内なので自動削除されません。参照目的以外では不要です。

backup/original_docs/ および doc/source/

プロジェクト設計時の検討資料・メモです。doc/source/ は現在の公式ドキュメント群、backup/original_docs/ は過去バージョンのドキュメントです。コードには影響しません。

ルートの 1567_14913.ltjruby / 789_14547.ltjruby

過去のビルド実行時にルートディレクトリに生成された中間ファイルです。 make clean で削除できます。

static/color-palettes.json

src/aozora_server.py と Web UI が使用する配色パターン定義ファイルです。 GET /api/colors エンドポイントがこのファイルを読み込みます。色カテゴリと明色・暗色リストが定義されており、コントラスト比 4.5:1 以上の組み合わせのみが API レスポンスに含まれます（WCAG 2.1 AA 基準）。

static/docs/ 内の HTML ドキュメント

Flask サーバー経由で /docs にアクセスすると読めるマニュアルです。開発者向けの参照資料として用意されています。コードには影響しません。

backup/original_docs/reference/ および static/docs/color|device/

配色・デバイスサイズの参照用 HTML ファイルです。src/aozora_server.py の /colors.html と /device-paper-size-map.html のルートから配信されています。

12. トラブルシューティング

Python パッケージのインストールが失敗する

# pip をアップグレードしてから再試行
python -m pip install --upgrade pip
pip install -r requirements.txt

lxml のインストールでエラーが出る

Windows では lxml のビルド済みホイールが pip から取得できなくなる場合があります。

pip install lxml --only-binary :lxml:

HTML が正しく変換されない

青空文庫の XHTML は <?xml ...?> 宣言を持つため lxml パーサーで警告が出ることがあります。

# XML パーサーを明示的に指定
python -m src.aozoratex data/xxx.html --parser lxml-xml

フォントが見つからない

settings.ini の font を確認し、システムにインストール済みのフォント名に変更してください。

# インストール済みフォントの確認（PowerShell）
[System.Drawing.FontFamily]::Families | Select-Object Name | Where-Object { $_.Name -like "*Mincho*" }

PDF のページ番号がズレる

LuaLaTeX は 2 回コンパイルする必要があります。1 回のみだと \pageref{LastBodyPage} が未解決になります。

make コマンドが見つからない

Windows では make は標準インストールされていません。以下の方法で入手できます。

  • Chocolatey: choco install make
  • Scoop: scoop install make
  • Git for Windows: Git Bash に同梱

make を使わない場合は Makefile の各ターゲットを直接 python コマンドで実行してください。

13. 拡張・開発ガイド

新しいデバイスを追加する手順

  1. settings.ini に新しいセクションを追加:
[PDF_newtablet]
font = Yu Mincho
font_size = 12pt
characters_per_line = 38
line_spacing = 1.0
newtablet_width = 190
newtablet_height = 260
  1. src/aozoratex.py の DEVICE_SIZES 辞書にフォールバック値を追加（任意）:
DEVICE_SIZES: dict[str, tuple[float, float]] = {
    ...
    "newtablet": (190.0, 260.0),
}
  1. src/aozoratex.py の argparse の choices リストに追加: 
choices=[..., "newtablet"],
  1. src/aozora_server.py の api_devices() 関数に追加:
"newtablet": {"width": 190, "height": 260, "label": "New Tablet (190×260mm)"},

配色パターンを追加する

static/color-palettes.json を編集します。palettes の各カラーには c（カラーコード）と n（名前）が必要です。コントラスト比が 4.5:1 以上の組み合わせだけが自動的に選択されます。

変換ロジックを拡張する

src/aozoratex.py の html_to_latex_body() 関数が本文変換の中心です。青空文庫に固有の HTML 要素（特殊クラス名など）を変換したい場合はこの関数内に処理を追加します。

テスト方法

現在のフォーマルなテストスイートはありません。make test / make test-all でサンプルファイルを用いた動作確認ができます。

make test       # iPhone 用変換のみ（高速確認）
make test-all   # 全デバイスで変換（包括確認）
