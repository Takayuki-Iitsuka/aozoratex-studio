プロジェクト要件およびシステム仕様書

1. 目的

青空文庫の HTML ファイルから LuaLaTeX を用いて縦書き・高品質な PDF を生成するツールを開発する。各デバイス向けに最適化された PDF を提供し、読みやすさと没入環境を追求する。

2. システム・アーキテクチャ要件

  • コア言語: Python（DOM 解析、ファイルI/O、システムコール、ローカルサーバー構築）
  • バックエンド（ローカルサーバー）: カスタマイズ用フロントエンドのエンドポイントとして、Pythonの軽量フレームワーク（Flask または FastAPI 等。※要選定）を使用してローカルサーバーを立ち上げる。
  • フロントエンド:
    • colors.html 等の HTML/JS/CSS で UI を構築する。
    • サイズや背景色・文字色のカスタマイズを容易にする。
    • デバイスに応じた用紙サイズ選択に device-paper-size-map.html を活用する。
  • コンパイラ: LuaLaTeX (lualatex)。必要に応じて Lua スクリプトを活用。
  • ビルドツール: 状況に応じて make や Python の subprocess でパイプラインを組む。
  • 設定ファイル: 端末別設定、フォント設定などは settings.ini に分離しカスタマイズを容易にする。

3. 入力データ仕様とエンコーディング方針

公式ドキュメント（https://www.aozora.gr.jp/KOSAKU/textfile_checklist/）に準拠したテキスト版（Shift_JIS + CRLF）の存在も考慮するが、HTML/XHTML を優先的に入力に用いる（ルビや構造が DOM ツリーから取得できるため）。

  • 文字コードの正規化: 入力ファイルから読み込む際、内部表現を必ず UTF-8 (Pythonの str) に変換してから扱う。
  • 外字と注記ルール: 青空文庫の公式注記一覧・外字注記辞書に従い、LaTeX 用のコマンドや画像埋め込みに変換する実装を構築する。

4. PDF 出力の対象

以下の4種類のデバイスサイズに分岐して出力できるようにする。

  1. PC用: A4用紙（白背景・黒文字）。上下2段組レイアウトで読みやすくする。
  2. iPhone11用: 実寸サイズ（カスタマイズ配色）。
  3. Android（スマホ）用: モダンサイズ（カスタマイズ配色）。
  4. iPad用: 横向き見開き2段組サイズ（カスタマイズ配色）。

5. TeX ファイル（テンプレート）の構造と組版ルール

出力される TeX コードのメンテナンス性を高めるため、出力コンポーネントを3層に分離する。 1. 表紙 (Title Page): タイトル、著者名等。 2. 本文ページ (Body Content): 青空文庫テキストのパース内容。 * ページスタイルは「書籍用レイアウトの横線（headings）」を廃止し、通常スタイル（番号を下部中央に表示する plain）を使用する。 3. 最終ページ (Colophon / 奥付): 青空文庫の HTML 最終ページにある原典情報等も LaTeX 化して流し込む。

6. コーディング規約

以後の拡張性（フォント追加や新規テーマ追加など）を考慮し、カスタマイズが容易になるよう、プログラミング言語に詳細なコメントブロックを持たせて実装すること。
