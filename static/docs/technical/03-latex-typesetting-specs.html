LaTeX組版_変換仕様

このドキュメントでは、青空文庫のテキストおよびHTMLデータを、jlreq クラスを用いた LuaLaTeX で美しい縦書きPDFに変換するためのシステム開発方針と、ページスタイル等のレイアウト設定基準を定義します。

変換パイプラインのアーキテクチャ

Windows環境で「青空文庫データ → TeX → PDF」の全自動ビルドを行うための設計方針です。

  1. HTML/XHTMLの入力を優先 テキスト版（Shift_JIS＋注記）からの正規表現パースはエッジケースが多く破綻しやすいため、HTMLのタグ構造（ルビや各種見出し要素を含む）を利用して DOM ツリーから TeX へ写像（Mapping）します。
  2. 文字エンコーディングの正規化 ファイル読み込み時に、内部文字列を必ず UTF-8（Pythonの str）に変換します。TeX エンジンは LuaLaTeX を使用するため、出力される .tex ファイルは常に UTF-8 となります。
  3. TeXテンプレートへの流し込み 抽出した本文データを、Python 等のテンプレートエンジン（Jinja2 など）を用いて、デバイス別（PC, Android, iPhone, iPad）のプリアンブル（設定）と結合します。

主要な写像ルール（HTML → TeX）

1. 段落と改行

  • <p>: 空行による段落生成、または \par コマンドに置換
  • <br>: 基本は段落整形を優先。必要であれば \\ を出力

2. ルビの処理

HTML5の <ruby> タグをパースし、luatexja-ruby で処理するためのコマンドへと変換します。 - ruby 内の <rp>（括弧）は破棄 - <rb>（親文字）と <rt>（ルビ文字）を抽出 - 出力先コマンド: \ltjruby{親文字}{ルビ}

3. 見出し

青空文庫の h1 〜 h6 タグや対応する CSS クラスを、LaTeX の階層構造に変換します。 - 大見出し: \section{...} - 中見出し: \subsection{...} ※ 実際の階層レベルは出力テンプレートの設定に依存します。

4. 傍点・傍線・外字

テキストの強調については、最低限の実装からスタートし、段階的に拡張します。 - 傍点: \textbf{}（太字）等で仮実装し、のちに対応パッケージへの変換をサポート。 - 外字（Gaiji): HTML内で <img class="gaiji"> として扱われるものをパースし、可能な限りUnicodeのテキストへ置換します。置換不可能な場合は画像をそのまま埋め込みます。

5. エスケープ処理（必須事項）

LaTeXのコンパイルエラーを防ぐため、本文中の以下の予約文字は、変換スクリプト内で必ずエスケープしてください。 - # $ % & _ { } \ ^ ~

LaTeX ページレイアウト（Page Style）の設定

青空文庫の PDF 化において、「上部の横線（Header rule）を消し、ページ番号（Folio）を下部中央に配置する」という要望を実現するための組版設定です。

設定値

LaTeX の標準機能である \pagestyle を変更します。jlreq クラスのデフォルト（headings）から、小説・文庫本の標準である plain へ切り替えます。

% Preamble ( \begin{document} の前 ) に記述
\pagestyle{plain}

適用時の挙動と影響

  • 上部横線の非表示: 章や節の見出し、およびヘッダーの横線が完全に非表示となり、文字領域のみの静かな読書空間が確保されます。
  • ページ番号の中央化: 奇数・偶数ページによる左右の振れ（小口寄せ）がなくなり、PDF Viewer での「単一ページ表示」時に番号の位置が完全に固定されます。
  • 注意点: plain スタイルでは現在読んでいる「章（Chapter）」のタイトル表示が消えるため、現在位置を把握するためのナビゲーション能力が下がります。
