開発者向け説明書

AozoraTeX Studio の現行実装（Next.js App Routerフロントエンド ＋ Pythonブリッジ・バックエンド）を前提にした詳細な運用・開発ガイドです。

1. システムアーキテクチャの概要

当システムは、UIとAPIを提供するNext.js (Node.js/Bun)と、実際のファイル処理やTeX生成を担うPythonプログラムが連携するハイブリッド構成をとっています。

  • フロントエンド (Next.js): React 19, Tailwind CSS v4, Framer Motion を用いたモダンなSPAとして構築。
  • Web API (Next.js API Routes): src/app/api/ 配下でリクエストを受け付け、Pythonのサブプロセスを呼び出します。
  • バックエンド連携 (Python Bridge): src/api_bridge.py が Next.js からのJSON入力を標準入力等で受け取り、適切なPythonモジュール（src/server_services.py等）に処理をディスパッチします。
  • コア変換エンジン (Python): src/aozoratex.py が HTML/XHTML をパースし、LuaLaTeX 向けの .tex に変換します。

2. 前提環境・開発ツールの要件

項目 要件 補足・推奨
Node.js / Bun Node v20+ または Bun UIのビルド・開発サーバー実行に使用。パッケージ管理は bun install または npm install
Python 3.11 以上 パッケージ管理・仮想環境には uv の使用を強く推奨。
TeX Live LuaLaTeX が利用可能 PDF 生成には latexmk と lualatex（LuaHBTeXなど）が必要。

3. クイックスタート (開発環境構築)

# 1. Python仮想環境の構築 (uvを使用する場合)
uv venv .venv
uv pip install -e ".[dev]"

# (標準の pip を使用する場合)
# python -m venv .venv
# .venv\Scripts\Activate.ps1
# pip install -e ".[dev]"

# 2. Node依存パッケージのインストール
bun install  # または npm install

# 3. 開発サーバー起動
bun run dev  # または npm run dev, start_server.bat などを利用

ブラウザで http://localhost:3000 を開きます。

注: 以前のFlaskサーバー (aozora_server.py) は完全に廃止され、Next.js API Routes + Pythonサブプロセスブリッジへと移行完了しています。

4. ディレクトリ構成

aozoratex-studio/
├── src/
│   ├── app/                      # Next.js App Router (フロントエンド UI + バックエンド API Routes)
│   ├── components/               # Reactコンポーネント群 (Tailwind, framer-motion)
│   ├── lib/                      # フロントエンドの各種フック・ユーティリティ
│   ├── api_bridge.py             # 【超重要】Node.js から呼ばれるPythonディスパッチャ (CLI/JSON I/O)
│   ├── aozoratex.py              # HTML/XHTML -> .tex 変換コアロジック
│   ├── server_services.py        # PDF生成・コンパイル・色/フォント管理サービス
│   ├── settings_store.py         # INI/Pydantic を利用した設定の読み書き・バリデーション
│   └── templates/                # 変換時に埋め込まれる各種Lua/Texテンプレート
├── config/
│   ├── global_settings.default.ini  # プロジェクト標準設定
│   ├── global_settings.custom.ini   # ユーザー上書き設定 (UIから変更可能)
│   ├── device_settings.default.ini  # デバイス標準レイアウト
│   └── device_settings.custom.ini   # デバイス別レイアウト上書き
├── static/
│   ├── docs/                     # 本ドキュメント等 (静的HTML配信)
│   └── ...
├── data/                         # 変換対象となる入力HTML/XHTMLサンプル
├── out/                          # 出力物 (.tex / .pdf の生成先)
├── pyproject.toml                # Pythonプロジェクト定義 (pydantic, ruff, mypy等)
└── package.json                  # Next.jsプロジェクト定義

5. データフローと連携の仕組み

  1. UI 操作: ユーザーがブラウザで変換対象ファイルやデバイス、テーマを選択し、生成ボタンを押下。
  2. Next.js API: src/app/api/generate/route.ts などがリクエストを受理。
  3. プロセス起動: Node.js (Next.js) が python src/api_bridge.py <コマンド> をサブプロセスとして非同期に spawn します。
  4. Python 処理実行: api_bridge.py は標準入力から設定やパラメータ（JSON）を受け取り、aozoratex.py や server_services.py を呼び出してTeXファイル生成と latexmk によるコンパイルを行います。
  5. 結果の返却 (SSE): Pythonプロセスの標準出力に流れるログを Next.js が Server-Sent Events (SSE) としてキャッチし、リアルタイムでUIへストリーミングします。

6. CLI 運用 (コマンドラインでの直接実行)

APIを経由せずに、Pythonプログラムを直接叩いて変換することも可能です。

python -m src.aozoratex_cli data/92_14545.html --device iphone --mode light --out out

出力先ディレクトリは統一されており、out/work/<device>/（TeX と中間生成物）および out/pdf/<device>/（PDF本体）へ出力されます。

7. Pythonモジュールのカスタマイズポイント

  • 組版・変換ロジックの調整: src/aozoratex.py 内のHTMLパース処理やTeXタグへの置換ルールを変更します。
  • 設定・バリデーションの拡張: src/settings_store.py に pydantic モデルを追加・修正することで、より堅牢な設定管理が行えます。
  • APIブリッジのコマンド追加: フロントエンドから新しいPythonの機能を利用したい場合は、src/api_bridge.py の DISPATCH テーブルにコマンドハンドラを新規登録します。

8. Next.js (フロントエンド) のカスタマイズポイント

  • UIコンポーネント: src/components/ 配下のReactコンポーネントを修正。Tailwind CSS v4によるユーティリティクラスを利用しています。
  • APIルートの追加: src/app/api/ 配下に新しいディレクトリ（[機能名]/route.ts）を作成し、Node.jsベースのバックエンド処理を実装します。
  • テーマとカラースキーム: UI自体のテーマ（ダーク・ライト）は next-themes を利用。PDF生成用のカラーリストは static/color-palettes.json など外部データを参照する仕様です。
