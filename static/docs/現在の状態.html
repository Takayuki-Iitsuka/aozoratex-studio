AozoraTeX Studio 現在の状態

最終更新日: 2026年（皇紀2686年／令和8年）7月4日

この文書は、プロジェクト内のソースコード、設定、依存ライブラリ、およびシステム全体のアーキテクチャの現在の確認結果を整理したものです。

1. LaTeX 関係

位置づけ

  • 当プロジェクトは、青空文庫形式のHTML/XHTMLを、LuaLaTeX（LuaTeX 系 LaTeX エンジン）向けの .tex ファイルに変換し、PDFへコンパイルするシステムです。
  • 中核となる .tex 生成処理は src/aozoratex.py が担っています。
  • PDFのコンパイルは latexmk を使用し、Webサーバー経由では src/server_services.py の _run_latexmk() によって実行されます。

実行環境

  • lualatex: LuaHBTeX 1.24.0 / TeX Live 2026（皇紀2686年／令和8年）
  • latexmk: バージョン 4.88 / 2026年3月9日版
  • コンパイルコマンドの基本は latexmk -pdf -lualatex を用いています。

文書クラスとパッケージ

  • 文書クラス: jlreq / tate (縦書き指定)
  • 主要パッケージ: luatexja, luatexja-fontspec, luatexja-ruby, graphicx, hyperref, bookmark, pdfcomment, xcolor, eso-pic, tikz (ライブラリ: calc), lltjext

2. システムアーキテクチャ (2026年最新構成)

フロントエンドとバックエンドの分離

  • 旧構成の廃止: 以前使用されていた Flask / SocketIO / WTForms による Python ベースの Web サーバー構成は完全に廃止されました。
  • フロントエンド (Next.js): ユーザーインターフェースと Web API の窓口は Next.js 15 (App Router) によって提供されています。
  • Python ブリッジ (API Bridge): Next.js の API Routes (src/app/api/**/route.ts) は、内部で Python の src/api_bridge.py をサブプロセスとして spawn し、JSONを標準入力/出力経由で受け渡しすることで処理を委譲します。
  • 非同期ログストリーミング: コンパイルの進捗ログなどは Python プロセスから Next.js へ送られ、Next.js からブラウザへは SSE (Server-Sent Events) を利用してリアルタイム配信されます。

3. Python 環境・構成

実行環境とパッケージ管理

  • Pythonバージョン: Python 3.11以上（現在 3.14.3 で動作確認済み）
  • パッケージ管理: pyproject.toml を用いた最新の標準構成へ移行済み。依存環境の構築には uv (例: uv sync または uv pip install -e ".[dev]") が推奨されています。

主要ソースファイル

  • src/aozoratex.py: 変換コアロジック (HTMLパース、LaTeXエスケープ、出力生成)。
  • src/api_bridge.py: Next.js からの呼び出しを受け受けるディスパッチャ (CLI/JSON I/O)。
  • src/server_services.py: ファイル一覧取得、latexmk 実行制御、フォント情報取得。
  • src/settings_store.py: 設定ファイルの読み書き (.ini 互換性を保ちつつ、pydantic でのバリデーションを導入)。

依存ライブラリ (pyproject.toml)

  • beautifulsoup4>=4.11.0, lxml>=4.9.0, markdown>=3.6 (HTML/マークダウン処理)
  • fontTools>=4.40.0 (フォントの外字カバレッジ解析等)
  • pydantic>=2.8, pydantic-settings>=2.3 (堅牢な型検証と設定管理)
  • 開発用: ruff, pytest, mypy

4. フロントエンド環境・構成

Web UI (Next.js)

  • エントリポイント: src/app/page.tsx
  • ポート: デフォルトで 3000 (http://localhost:3000)。起動は bun run dev または start_server.bat を推奨。
  • 使用技術: React 19, TypeScript, Tailwind CSS v4, framer-motion (アニメーション), lucide-react (アイコン)。

主な API エンドポイント

  • /api/data-files: 入力ファイル一覧の取得
  • /api/settings (GET/POST): 現在の設定の取得と保存
  • /api/generate: 単体PDFの生成要求（SSEによるリアルタイムログ出力）
  • /api/lualatex-fonts: 利用可能な LuaLaTeX フォント一覧の取得

5. 整理結果のまとめ (2026年7月)

  • UIの完全モダン化: Next.js (App Router) + React 19 + Tailwind CSS v4 により、堅牢でリッチなインターフェースを実現。
  • バックエンドの疎結合化: Flaskを排除し、Next.js API から Python ブリッジ (api_bridge.py) を呼び出す設計へ移行。
  • Pythonのエコシステム刷新: pyproject.toml によるパッケージ管理、uv の利用、pydantic による型安全な設定管理、ruff / mypy での品質保証。
  • 組版エンジンは堅調: LuaLaTeX + jlreq + TikZ による日本語縦書きPDF生成というコアの強みはそのまま維持。
