AozoraTeX Studio 現在の状態

作成日: 2026年（皇紀2686年／令和8年）7月3日

この文書は、project（プロジェクト）内の source code（ソースコード）、configuration（設定）、requirements（依存一覧）、static asset（静的資産）、および local environment（ローカル環境）の確認結果を整理したものです。

1. latex関係

位置づけ

  • この project（プロジェクト）は、青空文庫 HTML/XHTML を LuaLaTeX（LuaTeX 系 LaTeX エンジン）向けの .tex に変換し、必要に応じて PDF（Portable Document Format／携帯文書形式）へ compile（コンパイル）する構成です。
  • .tex 生成の中心は src/aozoratex.py です。
  • Web API（Web Application Programming Interface／Web アプリケーション用インターフェース）からの単体生成は src/aozoratex_generate.py を経由します。
  • PDF compile（PDF コンパイル）は latexmk を使います。Web server（Web サーバー）では src/server_services.py の _run_latexmk()、Makefile（メイクファイル）では LATEXMK_BASE と LATEXMK_FLAGS が担当します。

実行環境

  • lualatex --version の確認結果: LuaHBTeX 1.24.0 / TeX Live 2026（皇紀2686年／令和8年）
  • latexmk -v の確認結果: latexmk 4.88 / 2026年（皇紀2686年／令和8年）3月9日版
  • Makefile（メイクファイル）の compile command（コンパイル・コマンド）は latexmk -lualatex を基本にしています。
  • Web server（Web サーバー）の compile command（コンパイル・コマンド）は latexmk -pdf -lualatex を使っています。

LaTeX document class（文書クラス）と package（パッケージ）

src/aozoratex.py の template（テンプレート）では、次の document class（文書クラス）と package（パッケージ）を読み込んでいます。

  • document class（文書クラス）: jlreq / tate 指定
  • luatexja
  • luatexja-fontspec
  • luatexja-ruby
  • graphicx
  • hyperref
  • bookmark
  • pdfcomment
  • xcolor
  • eso-pic
  • tikz
  • TikZ library（TikZ ライブラリ）: calc
  • lltjext

組版・出力

  • device profile（端末別プロファイル）は smart、tablet、pc の 3 系統です。
  • 出力先は以下に整理されています。
  • .tex と intermediate file（中間ファイル）: out/work/<device>/
  • .pdf: out/pdf/<device>/
  • default font（既定フォント）は config/global_settings.default.ini の IPAmjMincho です。
  • layout（レイアウト）は config/device_settings.default.ini で管理されています。
  • smart は page number（ページ番号）を表示しない設定です。
  • tablet と pc は page number（ページ番号）を表示する設定です。
  • background（背景）と decoration（装飾）は TikZ（ティックズ）描画または image（画像）指定に対応しています。
  • cover texture（表紙テクスチャ）、washi background（和紙背景）、frame（枠装飾）を扱う実装があります。

2. python関係

実行環境

  • project local virtual environment（プロジェクト内仮想環境）: .venv
  • .venv\Scripts\python.exe --version の確認結果: Python 3.14.3
  • CLI（Command Line Interface／コマンドラインインターフェース）は .venv\Scripts\python.exe -m src.aozoratex_cli（または python -m src.aozoratex_cli）で動作します。
  • Web UI は Next.js 開発サーバー（bun run dev または npm run dev、ポート 3000）から提供され、内部で Python ブリッジを呼び出します。

主要 source file（主要ソースファイル）

  • src/aozoratex.py: HTML/XHTML parsing（解析）、LaTeX escaping（LaTeX エスケープ）、template rendering（テンプレート描画）、.tex 生成の中心（2841行規模のコアロジック）。
  • src/aozoratex_cli.py: CLI（コマンドラインインターフェース）の argument parsing（引数解析）と batch conversion（一括変換）。
  • src/aozoratex_generate.py: Web API および将来の batch 向け薄い facade（窓口）。
  • src/server_services.py: 生成オーケストレーション、latexmk 実行、フォント一覧、背景アセット、出力整理、color scheme 構築。
  • src/settings_store.py: .ini configuration（設定）読み書き（default/custom マージ）。
  • src/api_bridge.py: Next.js（Node/Bun）から Python ロジックを呼び出すための JSON ブリッジ（サブコマンド dispatch）。

requirements（依存一覧）

requirements.txt に記載されている direct dependency（直接依存）は以下です（Flask 系は移行により削除）。2026レビュー後: pyproject.toml + pydantic + ruff 推奨に移行中。

.venv に実際に入っている主な package（パッケージ）例

Next.js 側は package.json で管理（next, react, tailwindcss, framer-motion, lucide-react 等）。

test（テスト）

  • test file（テストファイル）は tests/test_aozoratex.py と tests/test_layout_settings.py です。
  • 主な test scope（テスト範囲）は heading bookmark（見出しブックマーク）、gaiji（外字）処理、空白保持、layout setting（レイアウト設定）、margin（余白）反映です。

4. サーバー / 実行環境

実行形態

  • Web UI は Next.js（bun run dev / npm run dev）によりポート 3000 で提供されます。
  • Next.js API Routes（src/app/api/**/route.ts）が runBridgeCommandSync / spawnGenerator を通じて src/api_bridge.py を呼び出し、Python の生成・設定ロジックを実行します。
  • Python 側に Flask は存在しません（移行完了）。コンパイルログは Server-Sent Events（SSE）でストリーミングされます。

起動方法

  • 推奨: start_server.bat
  • 手動: bun run dev または npm run dev
  • CLI: .venv\Scripts\python.exe -m src.aozoratex_cli ...

主な API エンドポイント（Next.js）

主なエンドポイントは以下です（/api/...）。

  • /api/data-files: 入力 HTML/XHTML 一覧
  • /api/background-assets: 背景画像一覧
  • /api/colors: 配色一覧
  • /api/lualatex-fonts: LuaLaTeX font（フォント）一覧（refresh 対応）
  • /api/devices: device setting（端末設定）
  • /api/settings (GET/POST): setting（設定）取得・保存
  • /api/settings/reset: custom setting（カスタム設定）初期化
  • /api/generate: 単体生成（SSE でログ＋結果ストリーム）
  • /api/session/organize: 出力整理
  • /api/session/cleanup-nonpdf: PDF 以外の中間生成物 cleanup（整理）

静的ドキュメント: /static/docs/... で提供。

port（ポート）

  • Web UI 既定ポートは 3000 です。
  • browser access（ブラウザアクセス）の案内は http://localhost:3000 です。
